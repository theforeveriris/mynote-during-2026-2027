## 1.物理层基本概念

物理层解决的是**如何在传输媒体上传输数据比特流**，而不是指具体的传输媒体（如光纤、双绞线）。传输媒体本身属于"第0层"。

物理层的基本特性包括：
- 机械特性
- 电气特性
- 功能特性
- 规程特性

| 特性       | 定义                  | 常见内容                  | 易混淆点                  |
| -------- | ------------------- | --------------------- | --------------------- |
| **机械特性** | 规定接口的物理外形、引脚排列、尺寸大小、接头结构、插拔方式等 | 接口形状（如 USB-A、RJ45、DB9）、引脚数量与排列顺序（如 VGA 是15针、USB 是4针）、接头尺寸、材质、抗压、插拔次数 | 看得见摸得着的物理结构           |
| **电气特性** | 描述信号在线缆上传输时的电压、电流、阻抗、传输速率、距离限制等物理电气参数 | 电平定义（逻辑 1/0 对应的电压范围，如 RS-232 中 +3V ~ +15V 为 0）、最大传输距离（如 RS-232 最大15米）、电缆特性阻抗（如 RJ45 为 100Ω）、最大信号频率/速率（如 USB 2.0 为 480 Mbps） | 是一个**范围**（如±10V~±15V） |
| **功能特性** | 规定每一根线的功能，即"这根线干什么用"，不同线路代表的控制/数据信号意义 | TXD（发送数据线）、RXD（接收数据线）、RTS（请求发送）、CTS（允许发送）、GND（信号地） | 是**含义**而非范围           |
| **规程特性** | 规定接口中信号的发送顺序、控制信号的配合方式、时序要求，即"谁先谁后、什么时候该做什么" | 握手流程（先 RTS，再 CTS，最后传输数据）、同步过程（时钟线、同步帧）、建立连接 → 传输数据 → 断开连接的顺序、起始位/停止位的时序要求（如串口） | 即"过程特性"               |

> 易混点：电气特性是"电压范围"，功能特性是"电平含义"。比如"用+10V~+15V表示0"中，前半句是电气特性，后半句是功能特性。


## 2.数据通信基础知识

**通信方式：**

| 方式      | 特点          | 信道需求 | 例子    |
| ------- | ----------- | ---- | ----- |
| **单工**  | 单向传输，无反向交互  | 1条   | 广播、电视 |
| **半双工** | 双方可收发，但不能同时 | 2条   | 对讲机   |
| **全双工** | 双方可同时收发     | 2条   | 电话    |

**数据传输方式：**

- 串行传输：逐位传输（单线，适合远距离）
- 并行传输：多位同时传输（多线，适合短距离）

**核心概念辨析**

| 概念      | 定义               | 单位   | 关系                |
| ------- | ---------------- | ---- | ----------------- |
| **码元**  | 固定时长的信号波形，代表离散数值 | —    | 1码元可携带多个比特        |
| **波特率** | 单位时间传输的码元个数      | Baud | 1 Baud = 1码元/s    |
| **比特率** | 单位时间传输的比特数       | b/s  | 比特率 = 波特率 × log₂V |
| **带宽**  | 单位时间最高数据率        | b/s  | 表示传输能力            |

> 关键公式：若码元有V种离散状态，则1码元携带 log₂V 比特信息。
> 二进制（2种）：1码元 = 1 bit
> 四进制（4种）：1码元 = 2 bits
> 八进制（8种）：1码元 = 3 bits
> 十六进制（16种）：1码元 = 4 bits

## 4.编码与调制（简要介绍，后续章节详细解释）

**数字数据 → 数字信号**

| 编码方式            | 特点             | 是否自带时钟      |
| --------------- | -------------- | ----------- |
| **非归零编码(NRZ)**  | 高电平1，低电平0      | 需外同步      |
| **归零编码(RZ)**    | 每个码元中间归零       | 自带时钟，但效率低 |
| **反向不归零(NRZI)** | 电平翻转表示0，不变表示1  | 无           |
| **曼彻斯特编码**      | 前高后低=1，前低后高=0  | 自带时钟      |
| **差分曼彻斯特**      | 码元开始处翻转=0，不变=1 | 抗干扰更强     |

以太网采用曼彻斯特编码

**数字数据 → 模拟信号（调制）**

| 调制方式           | 调制对象  | 特点          |
| -------------- | ----- | ----------- |
| **ASK** (调幅)   | 振幅    | 0无振幅，1有振幅   |
| **FSK** (调频)   | 频率    | 0低频，1高频     |
| **PSK** (调相)   | 相位    | 0和1对应不同相位   |
| **QAM** (正交调幅) | 振幅+相位 | 高频考点，同时调幅调相 |


**模拟数据 → 数字信号（PCM编码）**
 
PCM（脉冲编码调制）三步：

- 采样：按采样定理，采样频率 ≥ 2×最高频率
- 量化：将采样值分级取整
- 编码：将量化值转为二进制

> 采样定理：为了无失真恢复原始信号，采样频率必须 ≥ 2倍信号最高频率。

## 5.数据交换方式

| 交换方式     | 建立连接    | 传输特点       | 优点       | 缺点       |
| -------- | ------- | ---------- | -------- | -------- |
| **电路交换** | 需建立专用通路 | 独占资源，按序到达  | 时延小、实时性强 | 资源利用率低   |
| **报文交换** | 无需建立    | 存储转发，整报文传输 | 利用率高     | 时延大、需大缓存 |
| **分组交换** | 无需建立    | 存储转发，分组传输  | 利用率高、灵活  | 有额外开销    |

分组交换细分:

| 类型      | 特点           | 可靠性        |
| ------- | ------------ | ---------- |
| **数据报** | 无连接，每个分组独立路由 | 不保证顺序，可能丢包 |
| **虚电路** | 逻辑连接，分组沿同一路径 | 保证顺序，可靠性高  |
