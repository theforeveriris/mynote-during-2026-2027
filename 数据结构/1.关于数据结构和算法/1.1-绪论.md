## 1.数据结构的基本概念
- 数据
- 数据元素、数据项
- 数据对象、数据结构

数据是信息的载体，是描述客观事物属性的数、字符及所有能输入到计算机中并被计算机程序识别和处理的符号的集合。

## 2.算法的基本概念

> 程序 = 数据结构 + 算法

算法（Algorithm） 是对特定问题求解步骤的一种描述，它是指令的有限序列，其中的每条指令表示一个或多个操作。


算法的基本性质：

- **有穷性（Finiteness）**：一个算法必须总在执行有穷步之后结束，且每一步都可在有穷时间内完成。

- **确定性（Definiteness）**：算法中每条指令必须有确切的含义，对于相同的输入只能得出相同的输出。

- **可行性（Effectiveness）**：算法中描述的操作都可以通过已经实现的基本运算执行有限次来实现。

- **输入（Input）**：一个算法有零个或多个输入，这些输入取自于某个特定的对象的集合。

- **输出（Output）**：一个算法有一个或多个输出，这些输出是与输入有着某种特定关系的量。

- **正确性**：算法应能正确地实现预定的功能。

- **可读性**：算法应易于理解和交流。

- **健壮性**：当输入非法数据时，算法能适当地做出反应或进行处理，而不会产生莫名其妙的输出结果。

- **高效率与低存储量需求**：即时间复杂度和空间复杂度要尽量优化。

## 3.时间复杂度

如何评估算法时间开销？让算法先运行，事后统计运行时间吗？明显会存在许多问题：
- 和机器性能有关（超级计算机 vs 单片机）
- 和编程语言有关（越高级的语言执行效率越低）
- 和编译程序产生的机器指令质量有关
- 有些算法不能事后统计（如：导弹控制算法）

我们事前预估算法时间开销 T(n) 与问题规模 n 的关系（T表示"time"）

```
// 算法1 —— 逐步递增型爱你
void loveYou(int n) {      // n 为问题规模
    int i = 1;              // ① 爱你的程度
    while(i <= n) {         // ②
        i++;                // ③ 每次+1
        printf("I Love You %d\n", i);  // ④
    }
    printf("I Love You More Than %d\n", n);  // ⑤
}

int main() {
    loveYou(3000);
}
```

| 语句 | 执行次数  |
| -- | ----- |
| ①  | 1次    |
| ②  | 3001次 |
| ③④ | 3000次 |
| ⑤  | 1次    |

T(3000) = 1 + 3001 + 2×3000 + 1 = 9003, T(n) = 3n + 3

是否可以忽略表达式某些部分？如果有好几千行代码，按这种方法需要一行一行数？

| 表达式                       | 简化结果      |
| ------------------------- | --------- |
| T₁(n) = 3n + 3            | **O(n)**  |
| T₂(n) = n² + 3n + 1000    | **O(n²)** |
| T₃(n) = n³ + n² + 9999999 | **O(n³)** |

当问题规模 n 足够大时：
- 只考虑阶数最高的部分（结论1）
- 常数项系数也可以忽略（结论2）

## 4.空间复杂度

算法运行过程中 额外占用 的内存空间与 问题规模 n 之间的关系。
记作：S(n) = O(f(n))

```
void test(int n) {
    int flag[n];    // 这个数组长度与 n 相关，要算
    int i;          // 单个变量，与 n 无关，忽略
}
```

和时间复杂度一样，用大O表示法，只保留最高阶：S(n) = O(n)

| 代码                 | 空间分析          | 复杂度            |
| ------------------ | ------------- | -------------- |
| 只有 `int i` 和参数 `n` | 固定几个变量，与 n 无关 | **O(1)**（原地工作） |
| `int flag[n]`      | 申请了长度为 n 的数组  | **O(n)**       |
| `int flag[n][n]`   | 申请了 n×n 的二维数组 | **O(n²)**      |

空间复杂度 = 算法运行时额外申请的、与问题规模 n 相关的内存空间的 growth rate（增长趋势）。程序代码本身的大小不算，输入数据本身也不算，只算算法为了解决问题额外开辟的空间。

O(1) < O(log₂n) < O(n) < O(nlog₂n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)
