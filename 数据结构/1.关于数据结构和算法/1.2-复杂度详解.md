## 1.时间复杂度

| 复杂度        | 名称    | 典型场景        |
| ---------- | ----- | ----------- |
| O(1)       | 常数阶   | 数组访问、哈希查找   |
| O(log n)   | 对数阶   | 二分查找、循环变量翻倍 |
| O(n)       | 线性阶   | 单层循环遍历      |
| O(n log n) | 线性对数阶 | 快速排序、归并排序   |
| O(n²)      | 平方阶   | 双层嵌套循环、冒泡排序 |
| O(n³)      | 立方阶   | 三层嵌套循环、矩阵乘法 |
| O(2ⁿ)      | 指数阶   | 递归斐波那契      |
| O(n!)      | 阶乘阶   | 全排列、N皇后     |

### 1.1 O(1) 常数阶

核心：执行次数是固定常数，与 n 无关。

```
int getFirst(int arr[], int n) {
    return arr[0];          // 只执行1次
}

int sum = a + b;            // 1次
int x = 1;                  // 1次
x = x + 1;                  // 1次
```
无论 n 是 10 还是 10亿，只执行固定次数的操作。

## 1.2 O(log n) 对数阶

核心：循环变量按**倍数增长**（翻倍/减半），而不是逐个递增。

```
/循环翻倍
int count = 1;
while (count < n) {
    count *= 2;             // 每次乘以2
}
```

设执行了 k 次，则 count = 2^k，循环结束条件：2^k >= n

解得：k >= log₂n

```
/二分查找
int binarySearch(int arr[], int n, int target) {
    int left = 0, right = n - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;       // 每次搜索范围减半
    }
    return -1;
}
```
| 第几次 | 搜索范围      |
| --- | --------- |
| 1   | n         |
| 2   | n/2       |
| 3   | n/4       |
| ... | ...       |
| k   | n/2^(k-1) |

结束条件：n/2^k < 1，即 2^k > n，k > log₂n

### 1.3 O(n) 线性阶

核心：执行次数与 n 成正比

```
for (int i = 0; i < n; i++) {
    cout << i;              // 执行 n 次
}
```

### 1.4 O(n log n) 线性对数阶

核心：外层循环 n 次，内层操作 log n，或者递归深度 log n 每层处理 n 个。

```
/归并排序
void mergeSort(int arr[], int left, int right) {
    if (left >= right) return;
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);          // 递归左半
    mergeSort(arr, mid + 1, right);     // 递归右半
    merge(arr, left, mid, right);       // 合并，O(n)
}
```
> 递归深度：每次二分，深度 = log₂n
> 每层工作量：merge 合并操作，每层总共 O(n)
> 总工作量：深度 × 每层工作量 = log n × n = O(n log n)

### 1.5 O(n²) 平方阶

```
/双层循环嵌套
for (int i = 0; i < n; i++) {           // n次
    for (int j = 0; j < n; j++) {       // 每次内层n次
        cout << i << j;                 // 总共 n×n 次
    }
}
```

```
//内层依赖外层（冒泡排序）
for (int i = 0; i < n - 1; i++) {
    for (int j = 0; j < n - 1 - i; j++) {
        if (arr[j] > arr[j + 1])
            swap(arr[j], arr[j + 1]);
    }
}
```

> i=0 时，内层执行 n-1 次
> i=1 时，内层执行 n-2 次
> ...
> i=n-2 时，内层执行 1 次

> 总工作量：n-1 + n-2 + ... + 1 = n(n-1)/2 = O(n²)

### 1.6 O(n³) 立立方阶

```
/三层循环嵌套
for (int i = 0; i < n; i++) {           // n次
    for (int j = 0; j < n; j++) {       // 每次内层n次
        for (int k = 0; k < n; k++) {   // 每次内层n次
            cout << i << j << k;           // 总共 n³ 次
        }
    }
}
```

### 1.7 O(2ⁿ) 指数阶

核心：递归树呈指数级分支，大量重复计算。

```
//递归斐波那契数列
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);     // 每次分成2个分支
}
```
每层节点数大致翻倍
树的高度为 n
总节点数 ≈ 2⁰ + 2¹ + 2² + ... + 2ⁿ ≈ 2^(n+1) - 1
> O(2ⁿ) 

### 1.8 O(n!) 阶乘阶

核心：全排列问题，每个位置的选择数递减。

```
//N皇后问题
void solveNQueens(int n, int row, ...) {
    if (row == n) { ...; return; }
    
    for (int col = 0; col < n; col++) {   // 每行有n种选择
        if (isValid(...)) {
            board[row][col] = 'Q';
            solveNQueens(n, row + 1, ...);  // 递归下一行
            board[row][col] = '.';          // 回溯
        }
    }
}
```

| 行   | 可选列数       |
| --- | ---------- |
| 第1行 | n          |
| 第2行 | 约 n-1（剪枝后） |
| 第3行 | 约 n-2      |
| ... | ...        |

最坏情况下（无剪枝）：
T(n)=n*(n-1)*...*1 = n!

## 2. 时间复杂度计算

| 看到什么代码                     | 怎么算       | 结果         |
| -------------------------- | --------- | ---------- |
| 无循环，固定操作                   | 数语句条数     | O(1)       |
| 循环变量 `i *= 2` 或 `/= 2`     | 算需要几次到 n  | O(log n)   |
| `for i < n` 单层循环           | 执行 n 次    | O(n)       |
| `for i < n` 嵌套 `for j < n` | n × n     | O(n²)      |
| 三层嵌套                       | n × n × n | O(n³)      |
| 递归每次分成2份                   | 画递归树，数节点  | O(2ⁿ)      |
| 递归每层选择递减                   | 阶乘累积      | O(n!)      |
| 二分+每层处理n个                  | n × log n | O(n log n) |

顺序执行：O(f(n)) + O(g(n)) = O(max(f(n), g(n)))
嵌套循环：O(f(n)) × O(g(n)) = O(f(n) × g(n))


## 3. 空间复杂度

| 复杂度      | 说明     | 典型场景            |
| -------- | ------ | --------------- |
| O(1)     | 原地算法   | 冒泡排序、双指针、原地DP   |
| O(log n) | 递归栈空间  | 快速排序递归、二分查找递归   |
| O(n)     | 线性额外空间 | 归并排序、BFS队列、一维DP |
| O(n²)    | 二维结构   | DP表格、邻接矩阵、二维数组  |

### 3.1 O(1) 原地算法

核心：只用到固定数量的变量，不随 n 增长

```
// 冒泡排序
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];      // 只用1个临时变量
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```
> i, j, temp — 3个变量
> 无论 n=10 还是 n=100万，都只用这3个变量
> 在原数组上操作，不申请新数组

"原地" 的意思：就在输入数据所在的内存区域折腾，不另开新地盘。

### 3.2 O(log n) 递归栈空间

核心：递归调用需要栈空间保存每层的状态，深度是 log n

```
// 快速排序
void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pivot = partition(arr, low, high);  // 划分
        quickSort(arr, low, pivot - 1);         // 递归左
        quickSort(arr, pivot + 1, high);        // 递归右
    }
}
```
> 每层递归需要保存：low, high, pivot, 返回地址 等
> 递归深度：每次二分 → log₂n 层
> 最坏情况（数组已有序，每次划分极不平衡）：递归深度 = n，空间变成 O(n)

### 3.3 O(n) 线性额外空间

核心：申请了一个与 n 成正比的一维结构。

```
// 归并排序的临时数组
void mergeSort(int arr[], int left, int right) {
    if (left >= right) return;
    int mid = (left + right) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    
    // 合并时需要临时数组
    int* temp = new int[right - left + 1];   // ← 申请 n 大小的数组
    
    // ... 合并逻辑 ...
    
    delete[] temp;
}
```
> 每次 merge 都需要一个临时数组
> 虽然递归有多层，但各层的临时数组不会同时存在（用完就释放）
> 最大同时占用 = 最顶层 merge 的数组 ≈ n

```
// BFS 队列
void bfs(TreeNode* root) {
    queue<TreeNode*> q;
    q.push(root);
    
    while (!q.empty()) {
        TreeNode* node = q.front(); q.pop();
        if (node->left) q.push(node->left);
        if (node->right) q.push(node->right);
    }
}
```
> 队列最多存一层的节点
> 平衡二叉树最底层约 n/2 个节点

### 3.4 O(n²) 二维结构

核心：申请了 n×n 的二维数组或表格。

```
// 最长公共子序列
// dp[i][j] 表示 s1[0:i] 和 s2[0:j] 的 LCS 长度
int LCS(string s1, string s2) {
    int m = s1.length(), n = s2.length();
    int dp[m + 1][n + 1];           // (m+1) × (n+1) 的二维数组
    
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            // ... 填表 ...
}
```

```
// 邻接矩阵存图
// n个顶点，用n×n矩阵存边
int graph[n][n];    // graph[i][j] = 1 表示i到j有边

// 初始化
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        graph[i][j] = 0;
```

> 稀疏图可用邻接表优化到 O(n + m)，m 为边数


## 4. 空间复杂度计算

| 技巧          | 看到什么              | 空间复杂度       | 原理           |
| :---------- | :---------------- | :---------- | :----------- |
| **① 有限变量**  | `int i, j, temp`  | **O(1)**    | 固定个数，不随n增长   |
| **② 一维数组**  | `int arr[n]`      | **O(n)**    | 长度与n成正比      |
| **③ 二维数组**  | `int mat[n][n]`   | **O(n²)**   | 面积 n×n       |
| **④ 递归深度**  | 函数自己调用自己          | **O(深度)**   | 每层栈帧，看同时存在几层 |
| **⑤ 链表/树**  | `new Node` n次     | **O(n)**    | 每个节点固定空间     |
| **⑥ 哈希/集合** | `unordered_set`   | **O(n)**    | 存n个元素        |
| **⑦ 队列/栈**  | `queue` / `stack` | **O(最大宽度)** | 同时存的最大元素数    |
